# Future Extensions

Documents exploring potential improvements and features not yet implemented.

## Documents

### [ATTENDEE_EXTENSIONS.md](./ATTENDEE_EXTENSIONS.md)
Enhancements to the attendee/RSVP system based on RFC 5545 ATTENDEE properties:
- Invitations (organizer-created records)
- Enhanced RSVP with roles & delegation
- Capacity & waitlists
- Counter-proposals

### [ITIP_ACCESS_CONTROL.md](./ITIP_ACCESS_CONTROL.md)
Exploration of scheduling workflows and access control patterns:
- RFC 5546 (iTIP) concepts mapped to Pubky
- Calendar/event permission models
- Organizer vs Attendee authority
- Enforcement options in a decentralized system

---

## Status

These are **explorations**, not specifications. No decisions have been made.

The goal is to document:
1. What standards exist (RFC 5545, 5546, etc.)
2. How they might apply to Pubky's model
3. Trade-offs between approaches
4. Open questions requiring decisions
